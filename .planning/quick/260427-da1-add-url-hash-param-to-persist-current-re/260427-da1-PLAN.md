Add URL hash-based persistence of the active review step so that refreshing the browser
restores the user to whichever step (summary / walkthrough / review / submission) they
were on, instead of always resetting to 'summary'.

step flow. This is a paper-cut UX issue — the fix is two small surgical edits.

Output: Modified `web/src/store.ts` and `web/src/main.tsx`.

Two changes:

**1. `web/src/store.ts` — `setActiveStep()` (line ~328-331)**

After updating state and calling `emit()`, also set the URL hash fragment:

```typescript
setActiveStep(step: 'summary' | 'walkthrough' | 'review' | 'submission') {
  state = { ...state, activeStep: step };
  emit();
  location.hash = step;
},
```

This is fire-and-forget. No try/catch needed — `location.hash` assignment never throws.

**2. `web/src/main.tsx` — `bootstrap()` (line ~64-65)**

The `history.replaceState('', '', '/')` on line 65 wipes query params for T-03 token
leak mitigation. It also wipes the hash fragment. Fix by:

(a) Before `history.replaceState`, capture the current hash:
```typescript
const savedHash = location.hash; // e.g. '#walkthrough' or ''
```

(b) Change the replaceState URL to preserve the hash:
```typescript
history.replaceState('', '', '/' + savedHash);
```

(c) After the `openEventStream` call and before `createRoot`, validate and restore
the step. Define valid steps and parse:
```typescript
const VALID_STEPS = new Set(['summary', 'walkthrough', 'review', 'submission']);
const hashStep = savedHash.replace(/^#/, '');
if (VALID_STEPS.has(hashStep)) {
  actions.setActiveStep(hashStep as 'summary' | 'walkthrough' | 'review' | 'submission');
}
```

Place the `VALID_STEPS` set and the restore logic after `openEventStream(...)` but
before the `const rootEl = ...` block.

**What NOT to do:**
- Do NOT remove or weaken the `history.replaceState` — T-03 token leak mitigation
  must remain intact. Only the hash fragment is preserved, never the query string.
- Do NOT use `hashchange` event listeners or `popstate` — this is one-shot on load.
- Do NOT modify the INITIAL state default — it stays `activeStep: 'summary'`.
