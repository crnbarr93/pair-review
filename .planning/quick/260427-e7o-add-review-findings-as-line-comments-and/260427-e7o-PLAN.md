<objective>
Post self-review findings as inline GitHub comments alongside thread comments, and make the "Generate with Claude" review summary prompt more concise and verdict-focused.

Purpose: Currently only walkthrough Thread objects get posted as inline PR comments. Self-review findings (from run_self_review) have full anchor data (path, line, side, title, rationale) but are silently dropped during submission. This wastes the most valuable output of the review. The review summary prompt also produces overly verbose output when it should focus on verdict reasoning.

Output: Findings posted as inline comments on GitHub, findings included in local markdown export, summary generation prompt updated.
</objective>
