**Create `server/src/mcp/tools/submit-review.ts`**

Follow the `run-self-review.ts` analog exactly. Per D-05, this tool does NOT call Octokit — it proposes a review and returns immediately.

Key implementation details:
- **Tool name:** `'submit_review'` (D-13)
- **Description:** A detailed string explaining the two-step flow. Include: "Proposes a review for user confirmation. The review is NOT posted immediately — the user confirms or edits in the browser. GitHub mode: creates a PR review. Local mode: exports to a markdown file. The `exportPath` parameter is required for local-branch reviews (prKey starts with 'local:')."
- **Input schema (D-14):**
```typescript
const Input = z.object({
  prKey: z.string().min(1).max(200),
  body: z.string().min(0).max(65536),
  verdict: z.enum(['request_changes', 'comment', 'approve']),
  exportPath: z.string().optional(),
});
```
- **Session guard:** check `manager.get(prKey)` — return isError if null
- **Validation checks before proposing:**
  1. If `prKey` starts with `'local:'` and `exportPath` is not provided, return isError: `"exportPath is required for local-branch reviews"`
  2. Check that at least one thread has draftBody OR body is non-empty (no completely empty reviews): use `collectPostableThreads` from anchor.ts. If zero postable threads AND body is empty, return isError: `"No review content: body is empty and no inline comments have been drafted"`
  3. If `session.submissionState?.status === 'submitted'`, return isError: `"Review already submitted. submissionId: ${session.submissionState.submissionId}"`
- **Apply event:** `await manager.applyEvent(prKey, { type: 'submission.proposed', verdict, body })`
- **Return (D-15):** `{ content: [{ type: 'text' as const, text: 'Review proposed — awaiting user confirmation in the browser. The user will see the review summary, verdict, and inline comments in a submit modal. Once confirmed, the review will be posted to GitHub (or exported for local reviews).' }] }`
- **Error catch:** log via `logger.error`, return isError with message

**Create `server/src/http/routes/confirm-submit.ts`**

Per D-05, this is the browser-side POST endpoint that fires after user confirms in the modal.

Key implementation details:
- **Zod body schema:**
```typescript
const confirmSubmitBody = z.object({
  prKey: z.string().min(1),
  verdict: z.enum(['approve', 'request_changes', 'comment']),
  body: z.string().min(0).max(65536),
  exportPath: z.string().optional(),
}).strict();
```
- **Mount function:** `export function mountConfirmSubmit(app: Hono, manager: SessionManager): void`
- **Route:** `app.post('/api/confirm-submit', async (c) => { ... })`
- **Token middleware:** Automatic via `app.use('/api/*', tokenValidate(manager))` in http/server.ts — NO per-route middleware needed.
- **Handler logic:**
  1. Parse body with `confirmSubmitBody.safeParse` — return 400 on failure
  2. Get session via `manager.get(prKey)` — return 404 if not found
  3. Check `session.submissionState?.status === 'submitted'` — return 409 with `"Already submitted"` (D-10 idempotency)
  4. Generate `submissionId = nanoid(12)`
  5. Apply `submission.confirmed` event: `await manager.applyEvent(prKey, { type: 'submission.confirmed', submissionId })`
  6. Detect mode from `prKey` prefix:
     - **GitHub mode** (`prKey` starts with `'gh:'`):
       - Parse owner, repo, pullNumber from prKey (format: `gh:owner/repo/number`)
       - Call `submitGithubReview({ owner, repo, pullNumber, headSha: session.headSha, verdict, body, threads: session.threads ?? {}, submissionId })`
       - Apply `submission.completed` event: `{ type: 'submission.completed', reviewId: result.reviewId, url: result.url, submissionId }`
       - Return `c.json({ ok: true, url: result.url })`
     - **Local mode** (`prKey` starts with `'local:'`):
       - If `exportPath` is not provided, return 400 `"exportPath required for local reviews"`
       - Call `exportReviewMarkdown({ verdict, body, threads: session.threads ?? {}, baseRef, headRef, title, exportPath })`
         - Extract `baseRef`/`headRef` from session metadata (or from prKey if not available: `local:baseRef..headRef`)
         - `title` can be `session.pr?.title ?? 'Local Review'`
       - Apply `submission.completed` event: `{ type: 'submission.completed', submissionId, exportPath }`
       - Return `c.json({ ok: true, path: exportPath })`
  7. On any error in steps 6+:
     - Apply `submission.failed` event: `{ type: 'submission.failed', error: err.message }`
     - Return `c.json({ ok: false, error: err.message }, 500)`
