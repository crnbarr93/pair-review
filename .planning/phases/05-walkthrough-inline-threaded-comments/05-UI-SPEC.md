## Typography

Exactly four sizes, two weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (regular) | 1.45 |
| Label / secondary | 12px | 400 (regular) | 1.5 |
| Small / metadata | 11px | 400 (regular) | 1.2 |
| Code / mono | 12px | 400 (regular) | 1.55 |

Two weights only:
- **Regular (400):** all body, label, secondary, code, timestamp, and thread body text
- **Semibold (600):** headings, author labels in thread messages, sidebar section titles, brand wordmark, active walkthrough step label

**Phase-5-specific applications:**
- Walkthrough commentary banner body text: 13px / 400 / 1.5 (matches `.msg .body`)
- Walkthrough step list label (active step): 13px / **600** — use semibold for the active step to distinguish it; all other step labels at 13px / 400. Do NOT use 500 weight.
- Thread message author name: 12px / 600 (maps `.thread-msg .body .who` to the 12px label tier)
- Thread message body: 13px / 400 / 1.5 (maps `.thread-msg .body` to the 13px body tier)
- Thread timestamp: 11px / 400 / mono (maps `.thread-msg .body .time` to the 11px metadata tier)
- Post-body textarea (editable synthesis slot): 13px / 400 / 1.45 (matches existing `.chat-input textarea`)
- "N earlier messages" expander: 11px / 400 / `var(--ink-4)` color

**Source:** `web/src/index.css` — body rule + `.msg .body`, `.thread-msg` rules + `.chat-input textarea`.

## Color

All tokens are CSS custom properties on `:root`. Do not introduce new `--` variables in Phase 5. Map all new surfaces to existing tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--paper` `#FBFAF7` | App background, walkthrough banner background, thread card body background |
| Secondary (30%) | `--paper-2` `#F5F3ED` | Thread panel background (`#FFFDF7` — existing override), collapsed-turn background, walkthrough step list background |
| Accent (10%) | `--claude` `#2F6F66` | Curated-step left border, active walkthrough step indicator, thread marker dot (active thread), progress segment (done) |
| Destructive | `--block` `#9B3B2E` | Blocker-severity thread markers only |

Accent (`--claude`) reserved for exactly these Phase-5 elements:
1. Curated hunk left-border badge (3px solid left border on `.hunk` when it is a curated walkthrough step)
2. Active walkthrough step circle in StageStepper (inherits existing `.stage.active .num` — background `var(--ink)` not `--claude`; use `--claude` only for completed steps per existing `.stage.done .num`)
3. Thread marker dot in gutter (existing `.thread-marker { background: var(--claude) }`) for active/open threads
4. Post-body slot "Draft comment" action button border/tint — use `--claude-2` as background, `--claude` as text (matches existing `.chip.claude` pattern)
5. Thread left-border rule on `.thread-panel` (existing `border-left: 3px solid var(--claude)`)

Semantic color mapping for thread states:
- Open thread: `--claude` left border, `--claude` marker dot
- Resolved thread (no comment to post): `--ok` `#3A7A55` left border, `.thread-marker.resolved` (existing CSS)
- Blocker-severity thread: `--block` left border, `.thread-marker.blocker` (existing CSS)
- Warning/major thread: `--warn` `#B5791F` left border, `.thread-marker.warn` (existing CSS)

Walkthrough banner color:
- Container background: `#FFFDF7` (matches existing `.thread-panel` and `.summary-drawer` background — warm off-white that reads as distinct from `--paper`)
- Left accent border: 3px solid `var(--claude)`
- Body text: `var(--ink-2)` `#3A3832`
- Collapsed one-line summary: `var(--ink-4)` `#9A9585`

**Source:** `web/src/index.css` `:root` + `.thread-panel`, `.summary-drawer`, `.thread-marker.*`, `.stage.done .num` rules.

## Component Inventory (Phase 5 new surfaces)

All new components must use existing CSS classes where they exist, and add new CSS rules to `web/src/index.css` following the established Phase-N section comment pattern.

### WalkthroughBanner

Location in diff: above the hunk it narrates, pushed into the diff column (not the file explorer or findings sidebar).

Structure:
```
.walkthrough-banner            // new class — extends .thread-panel shape
  .walkthrough-banner-step     // "Step N of M" label — 11px mono, --ink-4
  .walkthrough-banner-body     // LLM commentary — 13px / 1.5 / --ink-2
  .walkthrough-banner-controls // Skip step button + "Mark visited" affordance
```

States:
- **Active step:** full height, `#FFFDF7` background, `3px solid var(--claude)` left border, body text visible
- **Previously visited (collapsed):** 32px height, `var(--paper-2)` background, one-line summary text in `var(--ink-4)`, no left border
- **Skipped:** same as collapsed, text shows "Skipped" in `var(--ink-4)`

CSS approach: add `.walkthrough-banner` and `.walkthrough-banner--collapsed` to the Phase 5 section in `index.css`. Reuse `.thread-panel` shape (padding, border-left, background) rather than duplicating.

### WalkthroughStepList

Location: replaces or augments StageStepper "Walkthrough" step area when walkthrough is active. Renders as a dropdown or inline sub-list under the "Walkthrough" stage item.

Structure: ordered list of step entries, each showing:
- Step number (mono, 11px, `--ink-4`)
- File + hunk header text (12px, `--ink-2`, ellipsized)
- Status chip: `done` (uses `.stage.done` styles), `active` (`.stage.active`), `skipped` (`--ink-4` text, no border fill)

The step list entry for the active step gets `border-bottom: 2px solid var(--ink)` (matches `.stage.active`). Active step label renders at 13px / 600 (semibold). All other step labels at 13px / 400.

Show-all toggle: a pill toggle button using the existing `.exp-toggle` pattern (inline-flex, `var(--paper-2)` background, `var(--line)` border, 5px border-radius). Two options: "Curated" | "All hunks". Active option uses existing `.exp-toggle button.on` treatment.

### ThreadCard (inline, push-down)

Renders as a `tr.thread-row` in the diff table — existing CSS slot already defined. The `td` inside uses `.thread-panel` as the styling base.

Structure inside `.thread-panel`:
```
.thread-panel
  .thread-panel .ref-loc        // "path:line · side" in mono, --ink-4
    .tag                        // severity or "thread" label — existing .tag / .tag.ok / .tag.blocker / .tag.warn
  .thread-msgs
    .thread-msg (× N turns)
      .av.claude / .av.me       // avatar — existing rules
      .body
        .who                    // author — 12px / 600 (label tier, semibold)
        .time                   // timestamp — 11px / 400 / mono (metadata tier)
        p                       // message text — 13px / 400 / 1.5 (body tier)
  .thread-older-expander        // "N earlier messages" — 11px / --ink-4 / text-btn style — new class
  .thread-draft-slot            // post-body textarea — appears after draft_comment is called
    textarea.thread-draft-input // new class — 13px / border: 1px solid var(--line-2) / border-radius: 6px
    .thread-actions             // existing class — buttons row
  .thread-actions               // resolve / close controls (when no draft)
```

Post-body slot (`.thread-draft-slot`): appears only after `draft_comment` MCP call sets the draft. Uses `var(--paper)` background, `1px solid var(--line-2)` border, `6px` border-radius — matching `.thread-reply`. Textarea is auto-resizing (min-height: 60px, max-height: 200px). Placeholder text: "Synthesized comment — edit before posting".

Collapse behavior: only the last 3 turns visible by default. Older turns hidden behind `.thread-older-expander`. Expanding shows all turns without animation (respects `prefers-reduced-motion` by default since no animation is added).

Thread marker in gutter (`.thread-marker`): existing CSS fully covers this. Shows count of turns when > 1 (e.g. "3"). Clicking scrolls to and expands the thread card. Diameter: 16px (corrected from prior 14px — see Spacing section).

### CuratedBadge

Applied to `.hunk` elements that are part of the curated walkthrough set.

Treatment:
- Add class `.hunk--curated` alongside `.hunk`
- CSS: `border-left: 3px solid var(--claude)` (existing focused hunk uses same but for navigation; curated badge is always-present when show-all is active)
- Optional "Curated" chip: 11px / mono / `var(--claude-2)` background / `var(--claude)` text / 3px border-radius — matches `.thread-panel .ref-loc .tag` pattern (use 11px small/metadata tier, not 10px)

Non-curated hunks in show-all mode: no badge, normal `.hunk` styles.

## Interaction Contracts

### Walkthrough navigation

Keyboard:
- `n` / `p`: advance/retreat within the curated walkthrough step list (when in curated mode) or all hunks (when show-all active). Existing Phase 3 keyboard handler extended.
- `c`: if focused hunk has an open thread, expand it; if not, show toast "Ask Claude to open a thread on this line" — this is the Phase 3 stub being wired.

Mouse:
- Clicking a walkthrough step in the step list scrolls to that hunk (reuses existing Phase 3 `scrollIntoView` + `focusedHunkId` pattern).
- Clicking "Skip step" on WalkthroughBanner marks step as skipped and advances to next.
- Clicking "Next step" / "Prev step" controls advances walkthrough cursor.

Show-all toggle: clicking "All hunks" renders all hunks interleaved with curated hunks badged. Clicking "Curated" snaps scroll to current walkthrough step. No page reload, no state reset.

### Thread interaction

Opening: thread-marker dot in gutter is clickable. Click expands the thread card (push-down row) below the anchored line. If thread card is already expanded, click scrolls to it.

Collapsing: clicking "×" or "Collapse thread" button on thread card collapses it to the gutter dot. Does not resolve the thread.

Resolving: "Resolve thread" button (`.btn-sm` style) sets thread state to resolved. Thread card background shifts to `var(--ok-bg)`, left border to `var(--ok)`, marker dot to resolved style. No confirmation required for resolve (non-destructive — resolves only local display state, nothing posted yet).

Post-body editing: textarea inside `.thread-draft-slot` is directly editable. No special save button — the textarea value is the current draft. On blur, the value is synced to local store state (not persisted via MCP — the draft body from `draft_comment` is already in the session store; UI edits are store-local until Phase 6 submission).

### Focus management

When walkthrough advances to a step: `scrollIntoView({ behavior: 'smooth', block: 'start' })` on the hunk element. Hunk receives `.focused` class for 400ms (existing Phase 3 animation). WalkthroughBanner expands to full height.

When thread-marker clicked: `scrollIntoView({ behavior: 'smooth', block: 'nearest' })` on the thread row. Thread card receives focus outline on the first interactive element (the close button or textarea).

## Layout Contract

**Main grid change for Phase 5:** The 2-column grid (`280px 1fr`) established in Phase 3 gains an optional third column when the walkthrough step list is in a dedicated panel. However, per CONTEXT.md D-17, FindingsSidebar already occupies a third column on `min-width: 1280px`. Phase 5 does NOT add a fourth column. The walkthrough step list renders inside the StageStepper expansion (below the "Walkthrough" stage tab), not as a new column.

```
grid-template-columns: 280px 1fr [280px when findings open]
grid-template-rows:    44px 52px 1fr
```

WalkthroughBanner renders inside the `1fr` diff column, above hunks — same column as the diff content.

Thread cards render inside the diff table rows — no layout column change required.

The StageStepper row (52px) may need to expand vertically when the walkthrough step list is open. If the step list opens as a dropdown/overlay below the stepper, it must use `position: absolute; z-index: 20` and not affect the grid row height.

## Accessibility

- WalkthroughBanner: `role="region"` `aria-label="Walkthrough step {N} of {M}"`. Active banner `aria-live="polite"` so screen readers announce step advances.
- Thread card: `role="region"` `aria-label="Thread on {path}:{line}"`.
- Thread marker (gutter dot): `role="button"` `tabIndex={0}` `aria-expanded={open}` (existing `ReadOnlyMarker` pattern extended).
- Post-body textarea: `aria-label="Draft comment body — edit before posting"`.
- Show-all toggle: `role="group"` + existing `.exp-toggle button` pattern (no additional ARIA needed, buttons are self-describing).
- Keyboard shortcut `c`: `aria-keyshortcuts="c"` on the diff canvas element.
- Thread older-messages expander: `aria-expanded={false}` when collapsed, `aria-expanded={true}` when expanded.
- `prefers-reduced-motion`: no new animations introduced; WalkthroughBanner collapse uses height transition only if the user agent permits (wrap in `@media (prefers-reduced-motion: no-preference)`).
