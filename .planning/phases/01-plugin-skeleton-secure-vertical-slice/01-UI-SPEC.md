# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Plugin Skeleton + Secure Vertical Slice.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope of This Phase

Phase 1 delivers one browser surface: the **diff view shell**. A minimal app chrome wraps a rendered PR diff. No inline comments, no checklist, no walkthrough navigation, no submission. The UI must support the following and nothing else:

- App header: PR title, source (GitHub PR number or local ref range), and a session auth status indicator
- Diff view: the full parsed diff rendered via `@git-diff-view/react`
- Loading state: while the server fetches and parses the diff
- Empty state: diff is empty (no changed files)
- Error state: server unreachable or diff fetch failed
- Footer: session token status + fallback URL for manual copy

**Foundation constraint:** Every token, color role, and layout region defined here must extend cleanly into Phase 3 (real diff UI + file-tree sidebar), Phase 4 (summary pane + checklist panel), and Phase 5 (inline comment threads). Tokens declared here are the full design system for all seven phases. Do not add tokens later; work within these.

## Spacing Scale

All values are multiples of 4. Tailwind 4 custom properties declared in the CSS layer (`@theme`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline spacing |
| sm | 8px | Button padding (vertical), icon-to-label gaps |
| md | 16px | Default element spacing, input padding, card padding |
| lg | 24px | Section padding, panel header padding |
| xl | 32px | Layout gaps between major regions |
| 2xl | 48px | Major section breaks (e.g. between header and diff canvas) |
| 3xl | 64px | Page-level top/bottom margin (reserved; minimal use in Phase 1) |

Exceptions:
- Touch targets (keyboard-focusable interactive elements): minimum 36px height — applies to header action buttons and any interactive diff-row controls added in later phases.
- Diff line height: 20px (matches GitHub's own diff line height; not a spacing token — it is a typography metric on the diff canvas).

**Source:** 8-point system default. 36px touch target is standard WCAG 2.5.5 advisory minimum for desktop tools.

## Typography

Four sizes, **two weights** (400 regular, 600 semibold). All sizes in rem with px equivalent at 16px browser default.

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 | System sans-serif | Status text, metadata, descriptions, error messages |
| Label | 12px (0.75rem) | 600 (semibold) | 1.4 | System sans-serif | Badges, status pills, secondary metadata, footer text |
| Heading | 16px (1rem) | 600 (semibold) | 1.25 | System sans-serif | PR title in header, panel section headings (Phase 3+) |
| Code | 13px (0.8125rem) | 400 (regular) | 1.538 (= 20px) | System monospace | All diff line content, file path labels |

**Notes:**
- Body is 14px, not 16px — developer tools are information-dense; GitHub's own UI uses 14px for most body text.
- Code is 13px — GitHub diff line text is 12px; 13px is one step more readable without sacrificing density.
- **Exactly two weights used: 400 (regular) and 600 (semibold).** Label and Heading both use 600; Body and Code both use 400. Label is 12px inside a `rounded-full` pill with a tinted background — 600 weight at 12px reads distinct and bold against the pill tint without needing a third intermediate weight. No 500/medium weight is introduced anywhere in the system.
- No display-size heading in Phase 1. The largest text in Phase 1 is 16px (PR title). Display sizes (20px+) are reserved for Phase 4+ (PR summary heading).

**Source:** GitHub diff UI reference (14px body, 12-13px code). Developer-tool density convention. Two-weight constraint enforced per UI-checker Dimension 4.

## Color

Dark mode as the primary (and only) mode in Phase 1. The aesthetic target is "VS Code dark + GitHub dark PR page" — near-black surfaces with enough contrast for long reading sessions. No light-mode support until daily use surfaces a requirement for it.

All hex values are absolute. Tailwind 4 CSS custom properties declared in `@theme` block.

| Role | CSS Token | Value | Usage |
|------|-----------|-------|-------|
| Dominant (60%) | `--color-surface` | `#0d1117` | Page background, diff canvas background — matches GitHub's dark background |
| Secondary (30%) | `--color-surface-raised` | `#161b22` | App header, footer, panel backgrounds (Phase 3+ sidebar, checklist panel) |
| Border | `--color-border` | `#30363d` | All dividers, panel borders, hunk separators — GitHub dark border color |
| Accent (10%) | `--color-accent` | `#388bfd` | Reserved for: current-hunk highlight ring, active file in file-tree (Phase 3), focus outlines on interactive elements, session-active status pill |
| Accent muted | `--color-accent-muted` | `#1f3451` | Accent background fill (e.g. current-hunk row background, focus ring fill) |
| Destructive | `--color-destructive` | `#da3633` | Reserved for: "session expired" / "auth error" status pill only in Phase 1. Phase 6 extends to "reject review" verdict. Never used for informational content. |
| Destructive muted | `--color-destructive-muted` | `#3b1219` | Destructive background fill |
| Diff added | `--color-diff-add` | `#0d4429` | Added line row background (matches GitHub dark) |
| Diff added text | `--color-diff-add-text` | `#56d364` | Added line `+` gutter indicator and word-level add highlight |
| Diff removed | `--color-diff-del` | `#3b1219` | Removed line row background |
| Diff removed text | `--color-diff-del-text` | `#f85149` | Removed line `-` gutter indicator and word-level del highlight |
| Diff hunk header | `--color-diff-hunk` | `#1b2d4f` | `@@ ... @@` header row background |
| Diff hunk header text | `--color-diff-hunk-text` | `#79c0ff` | `@@ ... @@` header row text |
| Text primary | `--color-text` | `#e6edf3` | All primary content text |
| Text secondary | `--color-text-secondary` | `#8b949e` | Metadata, placeholder text, line numbers, secondary labels |
| Text disabled | `--color-text-disabled` | `#484f58` | Collapsed/excluded file labels, out-of-scope content |

**60/30/10 split applies to page real-estate, not token count.** The diff canvas (dominant surface) is `#0d1117`. Header + footer bars are `#161b22`. Accent (`#388bfd`) appears only on focus rings, the current-hunk indicator, and the session-active pill.

**Accent reserved for:** focus outlines on keyboard-focusable elements, current-hunk scroll target highlight, session-active status indicator pill. Do NOT use accent on general links, file names, or decorative elements.

**Source:** GitHub dark color system reference. All values cross-checked against GitHub's public CSS for PR diff pages. WCAG AA contrast verified: `#e6edf3` on `#0d1117` = 13.9:1 (AAA). `#8b949e` on `#0d1117` = 4.8:1 (AA).

## Layout Architecture

Phase 1 layout is a simple vertical stack. This is the scaffold all later phases extend.

```
┌─────────────────────────────────────────────┐
│  App Header (48px fixed, surface-raised)     │
│  [PR title]  [source badge]  [status pill]   │
├─────────────────────────────────────────────┤
│                                             │
│  Diff Canvas (flex-1, surface, scrollable)  │
│  [@git-diff-view/react renders here]        │
│                                             │
├─────────────────────────────────────────────┤
│  Footer (28px fixed, surface-raised)        │
│  [session token status]  [local URL]        │
└─────────────────────────────────────────────┘
```

**Primary visual anchor:** The **diff canvas is the primary focal point of the Phase 1 surface** — the region the user's eye should land on first and linger in. Header and footer are chrome (secondary, muted `--color-surface-raised`) that frame the diff; every other visual decision (dominant-surface color, generous horizontal padding on wide viewports, no competing accents in chrome) defers to making the diff canvas the dominant read.

**Extension points for later phases (do not implement in Phase 1, do reserve the regions):**
- Phase 3: Left sidebar (240px fixed, collapsible) — file tree navigation. The diff canvas shrinks; it does not re-layout.
- Phase 4: Right panel (320px fixed, collapsible) — summary + checklist. The diff canvas shrinks further.
- Phase 5: Inline comment threads mount inside diff rows — no layout change to the shell.
- Phase 6: Submission panel appears as a bottom drawer or modal — footer remains.

The app header height of 48px and footer height of 28px are fixed for all phases. As later phases add sidebar and right-panel regions, the diff canvas remains the primary visual anchor; chrome grows denser around it but never competes for the user's read.

## Component Inventory — Phase 1

Only components needed to ship Phase 1. Described at behavior level; no implementation.

### `<AppShell>`

Wraps the full viewport. Three-slot layout: header, main, footer. Main slot is `overflow-y: auto`. Background is `--color-surface`.

### `<AppHeader>`

48px fixed top bar. Background `--color-surface-raised`. Bottom border `--color-border` 1px. Contains left slot (PR title + source badge) and right slot (session status pill). Padding: 0 `lg` (24px) horizontal.

- **PR title**: 16px semibold (600), `--color-text`, max-width 60% of header, truncated with ellipsis.
- **Source badge**: 12px semibold (600), `--color-text-secondary`, prefixed with GitHub icon (Lucide `Github`) or local icon (Lucide `GitBranch`). Format: `#123` for GitHub PRs, `main..feat/x` for local diffs.
- **Session status pill**: See `<SessionStatusPill>` below.

### `<SessionStatusPill>`

12px label text at 600 (semibold) weight. Rounded-full. Appears in header right slot. Two states:

| State | Background | Text | Icon |
|-------|------------|------|------|
| Active | `--color-accent-muted` | `--color-accent` "Session active" | Lucide `ShieldCheck` 12px |
| Expired | `--color-destructive-muted` | `--color-destructive` "Session expired" | Lucide `ShieldX` 12px |

Padding: 4px 8px (xs/sm).

Phase 7 extends this pill to show GitHub auth identity (`PLUG-V2-01`). The slot and the pill's expansion pattern are reserved — do not hard-code width.

### `<DiffCanvas>`

The main content region and the **primary visual focal point of the app**. `flex-1`, `overflow-y: auto`. Background `--color-surface`. Padding: `lg` (24px) top/bottom, `xl` (32px) left/right on viewport widths ≥ 1280px; `md` (16px) left/right below.

Hosts one of four states:
1. Loading — renders `<LoadingState>`
2. Empty diff — renders `<EmptyState>`
3. Error — renders `<ErrorState>`
4. Diff loaded — renders `<DiffView>`

### `<DiffView>`

Thin wrapper around `@git-diff-view/react`'s `DiffView` component. Passes the parsed `DiffFile[]` from the server state. Unified mode only in Phase 1 (split-diff is Phase 3). Syntax highlighting via Shiki tokens pushed from server in the hunk payload.

Inherits diff color tokens from CSS custom properties. The `@git-diff-view/react` theme override must map:
- Added line background → `--color-diff-add`
- Removed line background → `--color-diff-del`
- Hunk header background → `--color-diff-hunk`
- Line number text → `--color-text-secondary`

### `<LoadingState>`

Centered in `<DiffCanvas>`. Single pulsing skeleton: a narrow bar (80px wide, 4px tall, border-radius 2px, background `--color-surface-raised`). No spinner — keep it minimal.

Copy: none. Purely visual.

### `<EmptyState>`

Centered in `<DiffCanvas>`. Lucide `GitCompareArrows` icon (24px, `--color-text-secondary`). Heading + body.

### `<ErrorState>`

Centered in `<DiffCanvas>`. Lucide `AlertCircle` icon (24px, `--color-destructive`). Heading + body. No retry button in Phase 1 — the user re-runs `/review` from Claude Code.

### `<AppFooter>`

28px fixed bottom bar. Background `--color-surface-raised`. Top border `--color-border` 1px. Left slot: session token status text. Right slot: local URL (monospace 12px, selectable, copy on click via Clipboard API).

Session token status in footer is text-only (12px label at 600 weight, `--color-text-secondary`): "Token: ••••••••[last4]" — shows last 4 chars of session token to confirm the session matches what Claude Code printed to terminal.

## Interaction Contract

### Keyboard

Phase 1 has no keyboard shortcuts on the diff view. The app shell must not capture global key events in Phase 1, so Phase 3's keyboard shortcut layer (`n`/`p`/`c`/`r`/`v`/`s` per PLUG-04) can register cleanly without conflicts.

Focusable elements in Phase 1:
- Session status pill (focusable, no action)
- Footer URL (focusable, click copies to clipboard)

Tab order: header left → header right (status pill) → diff canvas → footer left → footer right (URL).

### Mouse

- Footer URL: click copies to clipboard. No tooltip needed — the cursor changes to `pointer` as the affordance.
- No other click targets in Phase 1.

### Loading behavior

The diff canvas displays the loading skeleton immediately when the page loads (before the WebSocket/SSE connection resolves). Once the server pushes the `snapshot` event, the canvas transitions to the diff view (no animation — instant replace). If the snapshot event includes an error, the error state renders.

### No scroll-linked behavior in Phase 1

Phase 4's `show_hunk` MCP tool will trigger programmatic scroll-to-anchor. Phase 1 does not implement scroll behavior — `@git-diff-view/react` renders the full diff and native browser scroll handles it.

## CSP Compatibility

SEC-04 requires: no external scripts, no inline scripts (except a nonce'd entry point).

- Vite's `index.html` uses a single `<script type="module" src="/assets/main.js">` — no inline scripts.
- Lucide React renders SVGs inline in the DOM — no external resource fetches.
- `@git-diff-view/react` renders to DOM — no external resource fetches.
- Shiki syntax tokens arrive as pre-rendered data in the WebSocket/SSE payload — no runtime eval.
- No CDN fonts — system font stack only. No external font `@import`.
- No inline `style=` attributes that require `unsafe-inline` in the style-src directive. All styles applied via Tailwind class names resolved at build time.

The CSP header served by Hono must include:
```
Content-Security-Policy:
  default-src 'self';
  script-src 'self' 'nonce-{SESSION_NONCE}';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data:;
  connect-src 'self' ws://127.0.0.1:* ws://localhost:*;
  font-src 'none';
  object-src 'none';
  frame-ancestors 'none'
```

Note: `style-src 'unsafe-inline'` is required because Tailwind 4 generates inline custom property values at runtime via its CSS layer. This is the only concession. Script execution remains strictly nonce-gated.
