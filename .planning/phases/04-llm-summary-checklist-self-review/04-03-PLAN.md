<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SessionEvent discriminator → reducer case | Discriminated-union narrowing is the type-safety gate. An unrecognized event.type would land on `default` and hit the `_never: never` guard → throw. |
| Event replay safety | Old snapshots on disk may lack summary/selfReview fields. Reducer operates only on events, not on pre-loaded state — the snapshot loader (manager.ts, unchanged) deserializes ReviewSession; reducer replays events after. Optional fields on ReviewSession (Plan 04-01) make pre-Phase-4 snapshots load cleanly. |
| Atomic replace semantics | summary.set / selfReview.set fully overwrite the field. No partial merge. No history. Per-prKey Promise-chain queue (Phase 2) already serializes sibling calls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-03-01 | Tampering | Reducer purity violation (accidentally touching lastEventId, mutating `s`, calling I/O) | mitigate | Unit tests assert output !== input (new reference), input.lastEventId === output.lastEventId, input.summary untouched when asserting output.summary. Grep-enforced: `grep -c "lastEventId" server/src/session/reducer.ts` returns 0. |
| T-4-03-02 | Denial of Service | Event replay replays a stale summary.set event after user regenerated | accept | Phase 2's replay is snapshot+events-since; since both events are atomic-replace, replaying a stale event merely restores an older summary — idempotent at the value level. Users regenerate manually (D-07). Not a DoS; a resume-to-old-state concern handled by the Phase 2 snapshot-on-write invariant. |
| T-4-03-03 | Information Disclosure | summary or selfReview containing LLM-authored text leaks to log output | mitigate | Reducer forbidden from logging (grep enforces 0 console.* calls in reducer). Logging lives exclusively at manager.ts / tool-handler layer. |
| T-4-03-04 | Tampering | New event variant breaks an existing reducer snapshot | mitigate | Optional `summary?: PrSummary | null` / `selfReview?: SelfReview | null` on ReviewSession (Plan 04-01) means old snapshots deserialize without the fields; reducer only writes them when events arrive. Verified by Plan 04-01 type additions + reducer test: "applying summary.set to a session with no prior summary returns a session with only the summary field added; all other fields preserved." |
</threat_model>
