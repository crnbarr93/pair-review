# Phase 06.3 Plan 02: Restyle ThreadCard and InlineComposer Summary

ThreadCard and InlineComposer restyled from inline styles to CSS class-based design with severity-colored header bars, avatar circles, and reply inputs matching the design.html mockup.

## What Was Built

**ThreadCard (D-06, D-07, D-08):**
- All inline styles removed; replaced with `tc-*` prefixed CSS classes
- Severity-colored header bar: `tc-header--blocker`, `tc-header--major`, `tc-header--minor`, `tc-header--nit`, `tc-header--neutral` — colored via `var(--block)`, `var(--warn)`, `var(--ink-4)` CSS variables
- Header displays severity label, line number, and finding category
- Avatar circles: `tc-avatar--claude` (24px, `var(--claude)` bg, "C") and `tc-avatar--user` (24px, `var(--ink-4)` bg, "Y")
- Message turns: `tc-turn`, `tc-turn-meta`, `tc-turn-name`, `tc-turn-time`, `tc-turn-body` classes
- Timestamps in HH:MM format using `tc-turn-time` class
- Reply input at bottom of unresolved threads: `tc-reply`, `tc-reply-input`, `tc-reply-btn`
- Reply submission via `postUserRequest` with `inline_comment` type
- `@claude` tag detection shows "Claude will respond" chip
- Draft slot preserved: `tc-draft-slot`, `tc-draft-input`
- Typing indicator preserved: `tc-typing` class
- Props extended: optional `finding?: ResolvedFinding` and required `prKey: string` added

**InlineComposer (D-09):**
- All inline styles removed; replaced with `ic-*` prefixed CSS classes
- `ic-panel` for container (replaces `thread-panel` with inline styles)
- `ic-textarea` for text area
- `ic-footer` for footer row (replaces `thread-reply-footer`)
- `ic-claude-chip` for @claude indicator (replaces `claude-chip`)
- `ic-btn-discard`, `ic-btn-submit` for action buttons (replaces `btn-sm`, `btn-sm primary`)
- `ic-footer-spacer` for flex spacer (replaces inline `style={{ flex: 1 }}`)
- @claude detection, keyboard handling (Enter/Escape), security comments all preserved unchanged

## Security

Both components maintain the security invariant: all LLM-authored content (`turn.message`, `draftBody`) is rendered exclusively as React text nodes — no `innerHTML` or `dangerouslySetInnerHTML` is used. Security comments documenting T-5-05-01 and T-5-05-02 are preserved in ThreadCard.

## Deviations from Plan

None — plan executed exactly as written. CSS rules for `tc-*` and `ic-*` classes will be added in Plan 04 Task 3 as specified.

## Known Stubs

None — both components are fully wired to `postUserRequest` for reply/comment submission.
