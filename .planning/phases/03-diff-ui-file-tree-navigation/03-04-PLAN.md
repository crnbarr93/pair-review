<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web → server POST /api/session/events | Client-supplied event body must include X-Review-Token header; fetch uses same-origin credentials |
| ReviewSession snapshot → AppState | Server-validated content; no transformation beyond field mirroring |
| CheckRun.link → `<a href target="_blank">` | Server-validated URL from `gh pr checks` JSON; rel="noreferrer" required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-05 | Tampering | postSessionEvent without CSRF token | mitigate | Helper function fails-fast with a throw if `reviewToken` is not set; always sends `X-Review-Token` header (grep-enforced). |
| T-3-08 | Spoofing | Malicious `CheckRun.link` URL | mitigate | The `<a>` tag uses `target="_blank" rel="noreferrer"` to prevent `window.opener` attacks. Link validation (http:/https:) happens at server boundary in Plan 03-02a via JSON parse + interface contract. |
| T-3-03 | Tampering | Displaying GitHub-supplied text (PR title, author, check names) | mitigate | All text renders as React text nodes via JSX interpolation (`{pr.title}`, `{check.name}`); no innerHTML path. Type system enforces string shape. |
| T-3-13 | Tampering | postSessionEvent called with undefined/empty prKey (would silently no-op the server route with 404) | mitigate | Plan 03-04 adds `prKey: string` to AppState and populates it from `msg.session.prKey` in onSnapshot + onUpdate. Plan 03-05's call sites early-return when `prKey === ''` (pre-snapshot bootstrap window). A store test asserts post-snapshot `state.prKey === msg.session.prKey`. |
</threat_model>

    ```typescript
    import type { SessionEvent } from '../../shared/types';
    // ... (existing imports unchanged; append SessionEvent to any existing type-only import group if that matches the file style)

    export async function postSessionEvent(
      prKey: string,
      event: SessionEvent,
    ): Promise<{ ok: true }> {
      if (!reviewToken) {
        throw new Error('postSessionEvent: review token not set — call setReviewToken first');
      }
      const res = await fetch('/api/session/events', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'X-Review-Token': reviewToken,     // double-submit CSRF — MUST be on every POST (T-3-05)
        },
        body: JSON.stringify({ prKey, event }),
        credentials: 'same-origin',
      });
      if (!res.ok) throw new Error(`postSessionEvent failed: HTTP ${res.status}`);
      return { ok: true };
    }
    ```

       ```typescript
       function CIPill({ ciStatus }: { ciStatus: CIStatus | undefined }) {
         const [expanded, setExpanded] = useState(false);
         if (!ciStatus || ciStatus.aggregate === 'none') return null;

         const palette: Record<CIStatus['aggregate'], { bg: string; fg: string }> = {
           pass:    { bg: 'var(--ok-bg)',    fg: 'var(--ok)' },
           fail:    { bg: 'var(--block-bg)', fg: 'var(--block)' },

           none:    { bg: 'var(--paper-2)',  fg: 'var(--ink-4)' },
         };
         const { bg, fg } = palette[ciStatus.aggregate];
         const failCount    = ciStatus.checks.filter(c => c.bucket === 'fail').length;
         const pendingCount = ciStatus.checks.filter(c => c.bucket === 'pending').length;
         const label =
           ciStatus.aggregate === 'pass'    ? 'All checks passed' :
           ciStatus.aggregate === 'fail'    ? `${failCount} check${failCount !== 1 ? 's' : ''} failing` :
           `${pendingCount} check${pendingCount !== 1 ? 's' : ''} pending`;

             style={{ background: bg, color: fg }}
       ```
