Keyboard spec (D-17 / D-18):
- Skip when `document.activeElement` is `INPUT`, `TEXTAREA`, or contenteditable.
- Skip when any meta/ctrl/alt modifier is held.
- `n` / `p` — advance focused hunk across cross-file virtual list of non-generated hunks; wrap with toast; scrollIntoView.
- `r` — toggle current file's review status (untouched|in-progress → reviewed; reviewed → in-progress). "Current file" is `focusedFileId` (set by n/p) OR first non-generated file if null.
- `c` → toast "Comments available in Phase 5"
- `v` → toast "Verdict picker available in Phase 6"
- `s` → toast "Submit available in Phase 6"

IntersectionObserver spec (D-11):
- `threshold: 0.5`, debounce 500ms.
- On entry crossing threshold, start 500ms timer. If still intersecting at t=500ms AND current status is `untouched`, POST `file.reviewStatusSet` with `status: 'in-progress'`.
- On exit crossing, clear the timer.
- Observer attaches to elements with `id={`diff-${file.id}`}`.

prKey spec (Plan 03-04 addition):
- Read `const prKey = state.prKey;` once in App body.
- Every handler that calls `postSessionEvent` MUST early-return when `!prKey` (covers pre-snapshot bootstrap window when the EventSource hasn't delivered a snapshot yet).
- Do NOT reconstruct `gh:${owner}/${repo}#${number}` — the server is the source of truth for prKey format; reconstruction silently fails for local-branch sessions where `pr.owner`/`pr.repo` are undefined.

Toast spec:
- Bottom-center pill, `role="status" aria-live="polite"`, 2500ms auto-dismiss.
- Implementation: `useState<string | null>` + `setTimeout` — no library.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| window keydown → client action | Client-owned; listener runs in user's browser with no privilege escalation |
| IntersectionObserver callback → POST | Client decides; server enforces via X-Review-Token + zod on /api/session/events |
| prKey carried to postSessionEvent | Must match server session; 404 otherwise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-05 | Tampering | postSessionEvent call path | mitigate | Plan 03-04 Task 1 already verifies X-Review-Token sent; App.tsx only calls the helper. |
| T-3-09 | Information Disclosure | Keyboard listener firing on INPUT (stealing keystrokes) | mitigate | Listener checks `document.activeElement.tagName` against INPUT/TEXTAREA and `isContentEditable`; returns early. Test case covers this. |
| T-3-10 | Denial of Service | IntersectionObserver storm on large PRs | accept | 50%/500ms debounce; each file fires once (server is idempotent for same status). Large-fixture render test (Plan 03-03) validates observer count is bounded to file count. |
| T-3-11 | Tampering | focused-hunk index out of bounds on diff reload | mitigate | `advanceHunk` always recomputes cross-file hunk list from current `state.diff.files`; stale index resets to 0. |
| T-3-13 | Tampering | postSessionEvent silent no-op when prKey is empty (pre-snapshot bootstrap) | mitigate | App.tsx reads `state.prKey` directly (Plan 03-04 store field), NEVER reconstructs from pr meta. Every call site has an explicit `if (!prKey) return;` early-return guard. Grep-enforced in acceptance criteria. |
</threat_model>

      const state = useAppStore();
      const diff: DiffModel | undefined = state.diff;
      // Plan 03-04 Task 2 added `state.prKey: string` and guarantees it is populated from
      // `msg.session.prKey` in both onSnapshot and onUpdate. Before the first snapshot,
      // state.prKey === '' — every postSessionEvent call site below early-returns on falsy.

      const [view, setView] = useState<DiffView>('unified');
      const [focusedHunkId, setFocusedHunkId] = useState<string | null>(null);
      const [focusedFileId, setFocusedFileId] = useState<string | null>(null);
      const [toast, setToast] = useState<string | null>(null);
      const focusedHunkIndex = useRef<number>(-1);

      // Build virtual hunk list, excluding generated files (D-18 + T-3-11: recompute from current diff each time)
      const virtualList = useMemo(() => {
        if (!diff) return [];
        return diff.files
          .filter(f => !f.generated)
          .flatMap(f => f.hunks.map(h => ({ fileId: f.id, hunkId: h.id })));
      }, [diff]);

      const showToast = useCallback((msg: string) => {
        setToast(msg);
        setTimeout(() => setToast(null), 2500);
      }, []);

      const advanceHunk = useCallback((delta: number) => {
        if (virtualList.length === 0) return;
        const current = focusedHunkIndex.current;
        // If stale (index out of bounds for new list), reset to 0
        const safeCurrent = current >= virtualList.length ? -1 : current;
        const wrapping = delta > 0
          ? safeCurrent === virtualList.length - 1
          : safeCurrent === 0 || safeCurrent === -1;
        const next = safeCurrent === -1
          ? (delta > 0 ? 0 : virtualList.length - 1)
          : (safeCurrent + delta + virtualList.length) % virtualList.length;
        focusedHunkIndex.current = next;
        const { hunkId, fileId } = virtualList[next];
        setFocusedHunkId(hunkId);
        setFocusedFileId(fileId);
        document.getElementById(hunkId)?.scrollIntoView({ behavior: 'smooth', block: 'start' });
        if (wrapping) showToast(delta > 0 ? 'Wrapped to first hunk' : 'Wrapped to last hunk');
      }, [virtualList, showToast]);

      }, []);

      // Global keydown listener (D-17)

          if (active && (active.tagName === 'INPUT' || active.tagName === 'TEXTAREA' || active.isContentEditable)) return;
          if (e.metaKey || e.ctrlKey || e.altKey) return;

            case 'p': e.preventDefault(); advanceHunk(-1); break;

        const timers = new Map<string, ReturnType<typeof setTimeout>>();
        const observer = new IntersectionObserver(

            for (const entry of entries) {
              const el = entry.target as HTMLElement;
              const fileId = el.id.startsWith('diff-') ? el.id.slice(5) : el.dataset.fileId;
              if (!fileId) continue;

                if (t) { clearTimeout(t); timers.delete(fileId); }
              }
            }
          },
          { threshold: 0.5 }
        );

          if (file.generated) continue;

          if (el) observer.observe(el);
        }
        return () => {
          observer.disconnect();
          timers.forEach(clearTimeout);
        };
      }, [diff]);

      }, [state.expandedGeneratedFiles]);

              onRequestChanges={() => handleCTAStub('Verdict picker available in Phase 6')}
