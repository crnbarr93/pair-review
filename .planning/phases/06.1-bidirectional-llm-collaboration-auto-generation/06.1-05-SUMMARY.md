# Phase 06.1 Plan 05: Slash Command Rewrite — Summary

**One-liner:** Rewritten `/pair-review` slash command orchestrates full bidirectional flow: auto-generates summary + walkthrough on startup, then enters indefinite `await_user_request` listen loop with all 12 MCP tools in `allowed-tools`.

## What Was Built

The `commands/pair-review.md` slash command was fully rewritten to implement the Phase 06.1 bidirectional collaboration model. The new command:

1. **Parses `$ARGUMENTS`** for `--dry` flag and strips it before building the source argument (three shapes: github URL, github number, `--local base head`).
2. **STARTUP SEQUENCE** — calls `start_review` → `set_pr_summary` → `set_walkthrough` in order (skips steps 2-3 if `--dry`).
3. **LISTEN LOOP** — enters an indefinite `await_user_request` loop. Handles all 6 return types: `no_request` (re-loops immediately), `chat` (calls `respond_chat` then loops), `inline_comment` (calls `reply_in_thread` + `draft_comment` then loops), `run_self_review`, `regenerate_summary`, `regenerate_walkthrough`.
4. **CRITICAL LOOP DISCIPLINE** section with emphatic imperative language — addresses T-06.1-14 (loop exit tampering threat).
5. **All 12 MCP tools** listed in `allowed-tools` frontmatter. Tool name cross-check confirms every name matches a `registerTool` call in `server/src/mcp/tools/*.ts`.
