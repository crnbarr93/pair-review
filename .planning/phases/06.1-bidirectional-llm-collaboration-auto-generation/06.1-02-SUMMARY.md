## Accomplishments

- Added `getActivePrKey()` to SessionManager with `activePrKey` field set on both the disk-resume path and the full ingest path in `startReview()`
- Created `await_user_request` MCP tool: fires `request.processing` event then long-polls for up to 5 minutes; returns `{ type: "no_request" }` on timeout and the request payload on delivery

- All security mitigations from threat model applied: zod `.strict()` schema (T-06.1-04), `resolveLineIdExtended` server-side validation (T-06.1-05), tokenValidate from /api/* glob (T-06.1-06)

## Decisions Made

- `getActivePrKey()` tracks the most-recently-started session rather than "current active" — idiomatic for single-user tool where only one review is active at a time
- `await_user_request` fires `request.processing` before `waitForRequest()` so the browser clears the pending indicator the moment Claude picks up the request, not when Claude responds
- `inline_comment` without `@claude` tag only fires `thread.userStarted` SSE (instant UI); no LLM queue entry needed for non-tagged user comments
- `chat` type fires both `chat.userMessage` applyEvent (instant SSE to UI) AND enqueues for LLM pickup — two effects for one POST are correct per D-05 design
