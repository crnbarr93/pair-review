## Decisions Made

- InlineComposer sends both tagged and non-tagged comments through the same postUserRequest endpoint with isClaudeTagged boolean — server (Plan 02 route) always fires thread.userStarted, but only enqueues to LLM queue when isClaudeTagged === true. Matches T-06.1-13 (accepted spoofing risk for single-user tool).
- Per-hunk state for hover/composer (useState inside UnifiedHunk/SplitHunk) rather than lifting to DiffViewer — keeps hunk components self-contained, avoids prop drilling for ephemeral UI state.
- SplitHunk InlineComposer uses representative line (pair.right.line ?? pair.left.line) so the lineId sent to the server is the most semantically useful (new-side preferred for add/context lines).
