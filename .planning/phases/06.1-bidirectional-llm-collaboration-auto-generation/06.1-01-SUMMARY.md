## Decisions Made

- **Thread.initiator generated by HTTP route (Option A from RESEARCH.md):** The HTTP route (`user-request.ts`, Plan 03) generates the threadId via nanoid and resolves lineId to (path, line, side) before firing `thread.userStarted`. The reducer receives all fields already resolved and just stores them. Keeps the reducer pure (no I/O, no nanoid).
- **D-14 draftBody conditional in reducer:** `thread.userStarted` with `isClaudeTagged: false` sets `draftBody = e.message` (user's review note becomes the draft GitHub comment body immediately). `isClaudeTagged: true` leaves `draftBody: undefined` (LLM synthesis via `draft_comment` required before submission).
- **enqueue clears resolver before calling it:** The `enqueue` method assigns `const resolve = this.resolver` then sets `this.resolver = null` before calling `resolve(req)`. This prevents a hypothetical second `enqueue` call during the resolve callback from double-resolving the same Promise.
