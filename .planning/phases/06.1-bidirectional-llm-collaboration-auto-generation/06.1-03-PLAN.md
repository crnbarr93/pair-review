**Part A: Add `postUserRequest()` to `web/src/api.ts`**

Add after the `postSessionEvent` function (end of file). Follow the `confirmSubmit` pattern exactly:

```typescript
export interface UserRequest {
  type: 'chat' | 'inline_comment' | 'run_self_review' | 'regenerate_summary' | 'regenerate_walkthrough';
  payload?: Record<string, unknown>;
}

export async function postUserRequest(
  prKey: string,
  req: UserRequest,
): Promise<{ ok: boolean; queued: boolean }> {
  if (!reviewToken) {
    throw new Error('postUserRequest: review token not set — call setReviewToken first');
  }
  const res = await fetch('/api/user-request', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Review-Token': reviewToken,
    },
    body: JSON.stringify({ prKey, ...req }),
    credentials: 'same-origin',
  });
  if (!res.ok) {
    const text = await res.text().catch(() => '');
    throw new Error(`postUserRequest failed: HTTP ${res.status} ${text}`);
  }
  return res.json();
}
```

  .chat-input (border-top, textarea + send button)
```

Props interface:

interface ChatPanelProps {
  messages: ChatMessage[];
  requestQueuePending: number;
  prKey: string;
  open: boolean;

}
```

Key implementation details:

1. **Message rendering** — SECURITY: all messages (user AND LLM) rendered as React text nodes. NEVER `dangerouslySetInnerHTML`. Use `<p style={{ whiteSpace: 'pre-wrap' }}>{msg.message}</p>`.

2. **Auto-scroll** — `useRef<HTMLDivElement>(null)` on `.chat-body`. `useEffect` on `[messages]` that auto-scrolls only if user is near bottom (`scrollHeight - scrollTop - clientHeight < 50`).

3. **Send handler** — Enter (without Shift) sends. Shift+Enter inserts newline. On send: call `postUserRequest(prKey, { type: 'chat', payload: { message: input.trim() } })`. Clear input immediately (optimistic). Catch errors and show a toast (`.toast` pattern from App.tsx).

4. **Queue badge** — When `requestQueuePending > 0`, show a badge in `.chat-head`: 20x20px circle, `--claude` background, white text, `--mono` 11px. `title="{N} message(s) queued"`.

5. **Run review button** — In `.chat-head`, right-aligned before collapse toggle. Label "Run review". `.btn-sm` class. On click: `postUserRequest(prKey, { type: 'run_self_review' })`. Disabled state when self-review is pending (if needed, track locally or derive from store).

6. **Empty state** — When `messages.length === 0`, show heading "Ask Claude anything" and sub-text "Review questions, code explanations, or ask Claude to focus on a specific area." with 3 suggestion chips: "Explain this change", "What are the risks?", "Regenerate walkthrough". Chips are buttons that call `postUserRequest` with type `'chat'` and pre-filled messages.

7. **Accessibility** — `aria-label="Message Claude"` on textarea. `role="log"` and `aria-live="polite"` on `.chat-body`. `aria-expanded={open}` and `aria-controls="chat-panel"` on collapse toggle. `aria-label="Start comment on line {N}"` on gutter icons (handled in Plan 04).

8. **Collapse toggle** — `<button>` with chevron-right SVG (or simple `x` text). `aria-label="Collapse chat"` / `"Expand chat"`. Calls `onToggle()`.

9. **Status indicator** — Show "Listening..." with a green dot (class `--ok`) when chat is open and no queue items. Show "Thinking..." with spinner when `requestQueuePending > 0`. These are informational UI elements, not persisted.

When `open` is false, render nothing (or a minimal collapsed bar with an expand button).
