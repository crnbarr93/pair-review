## Auto-generation flow

### What should auto-generate on startup?

| Option | Description | Selected |
|--------|-------------|----------|
| PR summary (set_pr_summary) | LLM reads the diff and generates the intent/key-changes/risk-areas summary | ✓ |
| Self-review (run_self_review) | LLM runs the criticality-ranked checklist to produce findings | |
| Walkthrough (set_walkthrough) | LLM picks core hunks and builds the narrative step order | ✓ |

**User's choice:** Summary + walkthrough, not self-review
**Notes:** Self-review is available on demand from the UI instead

### How should --dry mode work?

| Option | Description | Selected |
|--------|-------------|----------|
| Skip all auto-generation | --dry launches the browser with the diff loaded but no LLM-generated content | |
| Show diff + await UI requests | --dry skips auto-generation but the LLM stays in the loop waiting for UI-initiated requests | ✓ |

**User's choice:** Show diff + await UI requests

### Loop entry after generation

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-enter loop (Recommended) | Slash command instructs Claude to generate, then immediately start listening for UI requests | ✓ |
| User triggers loop | After generation, Claude reports done and waits for manual trigger | |

**User's choice:** Auto-enter loop

### Self-review from UI

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, UI button triggers it | A button in the UI lets the user request the self-review on demand | ✓ |
| No, terminal only | Self-review stays terminal-driven | |

**User's choice:** Yes, UI button triggers it

## UI → LLM channel

### Reverse channel mechanism

| Option | Description | Selected |
|--------|-------------|----------|
| Long-poll MCP tool (Recommended) | await_user_request() blocks until browser POSTs a request | ✓ |
| Periodic polling tool | check_inbox() every few seconds in a loop | |
| MCP sampling (createMessage) | Server requests completions from the client | |

**User's choice:** Long-poll MCP tool

### Queue behavior while Claude is busy

| Option | Description | Selected |
|--------|-------------|----------|
| Queue on server (Recommended) | Server queues incoming UI requests, Claude dequeues on next await_user_request call | ✓ |
| Reject while busy | Server returns 429 if Claude is currently processing | |
| Batch on next poll | Multiple requests accumulate and return in one call | |

**User's choice:** Queue on server

### Timeout behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Timeout with re-enter | Tool times out after ~5 min, returns no_request, Claude re-enters loop | ✓ |
| Block indefinitely | Tool blocks until user sends something | |

**User's choice:** Timeout with re-enter

### Chat scope

| Option | Description | Selected |
|--------|-------------|----------|

## @claude inline tagging

### User-initiated thread model

| Option | Description | Selected |
|--------|-------------|----------|
| User starts thread, LLM responds | User clicks line, types comment, LLM responds to all comments | |
| Always @claude required | Only @claude-tagged comments go to LLM. Non-tagged are user notes | ✓ |
| Hybrid: all go to LLM | Every user comment automatically sent to Claude | |

**User's choice:** Always @claude required — non-tagged are user notes

### Non-@claude comment behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-draft (Recommended) | Non-tagged comment immediately becomes the draft post body | ✓ |
| Raw note, manual draft | Comment sits in thread, user must manually finalize into draft | |

**User's choice:** Auto-draft

### Draft authorship for @claude threads

| Option | Description | Selected |
|--------|-------------|----------|
| LLM drafts via draft_comment | Same as Phase 5: Claude synthesizes thread into post body after conversation | ✓ |
| User writes their own | Claude responds but user writes the final draft manually | |
| Either can draft | User or Claude can create the draft | |

**User's choice:** LLM drafts via draft_comment

### Comment composer entry point

| Option | Description | Selected |
|--------|-------------|----------|
| Click gutter marker (+) | + icon in line number gutter on hover (GitHub PR style) | ✓ |
| Click line + keyboard shortcut | Click line to select, press 'c' to open composer | |

**User's choice:** Click gutter marker (+)
