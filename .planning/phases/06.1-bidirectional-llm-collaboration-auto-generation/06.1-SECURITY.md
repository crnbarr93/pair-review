## Accepted Risks Log

| Threat ID | Risk Statement | Rationale |
|-----------|---------------|-----------|
| T-06.1-07 | User-authored chat text returned to LLM context via await_user_request | Intended design for pair-review collaboration. User text flowing to LLM is the core product feature. No sensitive data boundary exists between user and their own LLM session. |
| T-06.1-08 | Client sends isClaudeTagged boolean without server re-verification of message text | Single-user tool with no multi-user trust model. Worst case outcome is the user's own comment being unnecessarily routed through the LLM queue. No escalation path. |
| T-06.1-13 | isClaudeTagged derived client-side from value.includes('@claude') | Duplicate of T-06.1-08 scoped to InlineComposer component. Same rationale applies. |
| T-06.1-15 | --dry flag can be injected via $ARGUMENTS in user's terminal | User controls their own terminal session. No elevation of privilege possible -- --dry only skips auto-generation, does not bypass any security control. |
