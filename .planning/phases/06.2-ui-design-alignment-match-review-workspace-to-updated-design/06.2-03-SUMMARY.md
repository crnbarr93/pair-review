# Phase 06.2 Plan 03: Component Inline Adaptation Summary

**One-liner:** Four components shed overlay/modal/collapsed behavior and become inline panels renderable inside RightPanel's content slot — ChatPanel always-open with context badge, FindingsSidebar with category chips, SubmissionPanel without modal wrapper, WalkthroughStepList with optional show-all.

## What Was Built

### Task 1: ChatPanel + FindingsSidebar adaptation (commit 50d0f67)

**ChatPanel.tsx:**
- Removed `open` and `onToggle` props — component is now always open inside the right panel
- Removed the collapsed state early-return block (the `if (!open)` branch rendering the expand button)
- Removed the collapse chevron button from the chat header
- Added `contextBadge?: string` prop — renders a `.chat-context-badge` pill between the header and message body (e.g. "SUMMARY", "WALKTHROUGH . auth.ts", "REVIEW . auth.ts . L42")
- Added `suggestionChips?: Array<{ label: string; message: string }>` prop with fallback to `DEFAULT_SUGGESTION_CHIPS`
- Changed root element from `<aside>` to `<div>` (no longer a standalone landmark)
- Removed `id="chat-panel"` and `aria-controls` attributes

**FindingsSidebar.tsx:**
- Removed `open` and `onClose` props — component renders inline whenever mounted
- Removed the `if (!open) return null` guard
- Removed the close button (×) from the header in both the empty state and the main render
- Added `findings-coverage-strip` with `coverage-chip` buttons (moved from StageStepper per D-09)
- Changed root element from `<aside>` to `<div>`

**index.css:** Added `.findings-coverage-strip` layout rule (flex wrap, gap, padding, border-bottom).

### Task 2: SubmitModal dual-mode + WalkthroughStepList (commit ad06b7f)

**SubmitModal.tsx refactored into three layers:**
1. `useSubmissionState()` — shared hook with all state, effects, and logic (verdict, body, retype gate, walkthrough completion check, submit handler)
2. `SubmissionContent({ onClose? })` — shared JSX tree; `onClose` absent = inline (no close/cancel buttons), present = modal (close + cancel buttons rendered)
3. `SubmissionPanel` export — wraps `<SubmissionContent />` in `.submission-panel` div; no self-guard; renders whenever mounted
4. `SubmitModal` export — legacy modal wrapper preserved; still self-guards on `submitModalOpen`; passes `onClose` to SubmissionContent; keeps Escape key listener

**WalkthroughStepList.tsx:**
- Made `onShowAllToggle` optional (`onShowAllToggle?: (showAll: boolean) => void`)
- No structural changes — component renders correctly in flex column right-panel context

**index.css:** Added `.submission-panel` rule (`overflow-y: auto; padding: 0`).
