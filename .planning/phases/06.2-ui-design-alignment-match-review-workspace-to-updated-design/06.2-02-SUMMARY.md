# Phase 06.2 Plan 02: TopBar Two-Row Header with StepNav Summary

**One-liner:** Two-row TopBar with PR info row + internal StepNav using numbered circles, chevrons, and clickable steps — StageStepper export removed.

## What Was Built

### Task 1: Rework TopBar into two-row header with StepNav

**TopBar.tsx** was fully rewritten:

- Outer `<header className="topbar-shell">` wraps two rows
- Row 1 (`.topbar`): brand, separator, PR info (owner/repo/number, title), branch display, CI pill, spacer, Settings button, Submit review button
- Row 2: internal `StepNav` component with 4 steps (Summary, Walkthrough, Review, Submit) connected by `Ic.chev` chevrons
- `StepNav` determines active step from `activeStep` prop (not data state) — every step is clickable via `onStepClick(key)`
- Sub-labels show contextual data: summary confidence %, walkthrough step counts, finding counts, submission state
- `StageStepper` named export removed entirely
- Category coverage chips (`.stages-coverage-strip`) removed per D-09
- Old props removed: `activeCategory`, `findingsSidebarOpen`, `onSummaryStep`, `onSelfReviewStep`, `onCategoryClick`, `onToggleFindingsSidebar`, `pendingSubmission`
- New simplified props: `activeStep`, `onStepClick`, `onSubmitReview`, `walkthrough` (added)
- `CIPill` component preserved unchanged

**index.css** updated: Added `.topbar-shell { display: flex; flex-direction: column; }` wrapper above the existing `.topbar` rule.

**TopBar.test.tsx** fully rewritten to match new interface:
- 13 tests covering: PR meta rendering, two-row structure, 4 step items, step labels, active step highlighting, CI pill show/hide, CI pill expansion, Settings callback, Submit callback, step click callback, submitted state display, aria-labels, no coverage strip

### Task 2: Verify icons.tsx has all needed icons

No changes made. All icons required by the two-row TopBar layout are already present:
- `Ic.chev` — chevron between steps
- `Ic.check` — completed step circle
- `Ic.branch` — branch display in PR info row
- `Ic.settings` — Settings button

The `arrowRight` and `refresh` icons mentioned in the plan are for the footer (Plan 01 / StepFooter scope), not needed here.
